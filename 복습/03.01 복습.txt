Day 56 복습 추천 구조
1️⃣ 상태 다이어그램 다시 그려보기
IDLE → RUNNING → CRITICAL_STOP → LOCKED → RESET → IDLE

IDLE → RUNNING: add_command (Immediate) 또는 update (Scheduled)에 의해 발생.

RUNNING → CRITICAL_STOP: 하드웨어 정지(Telescope.stop) 또는 명령 실패(CMD_FAILED) 감지 시 발생.

CRITICAL_STOP → LOCKED: update 루프에서 즉시 _is_critical = True로 전환되며 봉인.

LOCKED → IDLE: 오직 reset_critical() 호출을 통해서만 가능하며, 이때 ResetCommand가 하드웨어를 깨움.
